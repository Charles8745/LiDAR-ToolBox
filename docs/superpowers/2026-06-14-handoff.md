### 官方資料源備查 — KHB GetMarker

```
POST https://sdci.twport.com.tw/khbweb/osmx2.aspx/GetMarker
Body: {}
Response: { "d": "<JSON string>" }
  → JSON.parse(d).v  = occupied-vessel array
```

每筆 vessel 含:
- `PIER`:4 位數船席代碼
- `LAT1/LONG1`–`LAT2/LONG2`:船席兩端點座標(midpoint = 標籤位置)
- `ANGLE`:船席方位角(bonus,未規整化 mod 360;v1 未使用)
- `SP_NAME`:船席名稱

**重要特性**:
- 僅回傳**當前有船佔用**的船席(空泊位不出現)→ 需多次執行累積
- 從開發機可直接連線(**非**地理封鎖,不像 MPB AIS 需台灣 IP)
- 全台TWPort 資料 → 需 bbox-filter 至 KHH

## 6. 誠實邊界 / 已知小問題(別當成 bug 重修,除非要正式化)

- **是「泊位佔用隨時間」,不是連續 AIS 航跡**。船在佔用區間內出現於泊位、區間外消失;沒有逐秒移動。→ 這就是 Task 17「進港滑入動畫」被**刻意延後**的原因(做假動畫會誤導資料)。要真航跡需接 aisstream(免費需金鑰)。
- 碼頭座標是**描圖內插的近似**(順序對、絕對位置非測量級)。
- `MAX_BERTH=121`:資料中有 #122、#145(會被夾到南端;`rebuildIncoming` 只掃 1–121,這兩個的進港標記會略過)。改 `MAX_BERTH` 會位移所有泊位(別亂改),要修就讓 `rebuildIncoming` 改吃資料實際出現的 berthNo。
- 約 12 筆資料 `leaveMs < actPortMs`(髒資料),被 `Math.max(endMs, startMs+1)` 壓成 1ms 區間 → 那些船幾乎不出現。
- `occupancyAt` 重疊時「後者勝」依賴陣列順序(`[...berthing, ...forecast]` → 預報勝),非時間排序。規模小無妨。
- app 層 window 事件監聽無 teardown(單頁 example,無 dispose 路徑)。
